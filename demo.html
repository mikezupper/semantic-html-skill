The Semantic HTML Vocabulary

A self-demonstrating field guide — every element on this page is its own example

Published July 22, 2026 · Reading time about 12 minutes · Filed under HTML

This document is written under one rule: no CSS, no presentational JavaScript. What your browser renders is its default stylesheet applied to honest markup. Turn on a screen reader, or open the accessibility tree in devtools — the structure you hear is the structure you see, because the page is its structure.

Inline text semantics

The words you are reading sit inside a <p>. When a phrase carries stress that changes meaning, it takes emphasis; when it carries weight, it is strongly important. A keyword merely drawn to your attention uses <b>, and text in an alternate voice — like the taxonomic name Homo sapiens — uses <i>.

A term being defined, such as semantics (the meaning of markup, as opposed to its appearance), is a <dfn>. Abbreviations expand on hover: the W3C and the WHATWG maintain the specs. Titles of works take <cite>: HTML: The Living Standard.

Computer-speak has four distinct elements. Source fragments use document.querySelector(). Variables and placeholders use elementName. Program output uses Document is valid. 0 errors. Keyboard input uses Ctrl+Shift+I — note the nested <kbd> per key.

Machine-readable values: the product Field Guide, second edition ships on September 1. Search engines read the value and datetime attributes; humans read the prose. A search hit would be highlighted like this. Fine print is set with terms and conditions apply, chemistry with H2O, math with E = mc2, and a price that no longer applies is struck: $49.00 now $29.00.

Long unbroken tokens accept break opportunities with <wbr>: https://html-tags.com/guides/semantic-html/field-guide. A proper-noun mark in Chinese uses <u>, its only good use: 北京是中国的首都。

    Internationalization elements

    Every language switch is marked. She whispered c'est la vie and moved on. A German proverb: Übung macht den Meister — notice the quotation marks change with the language.

    East Asian pronunciation uses ruby annotations with parenthesis fallback: 東(とう)京(きょう) (Tokyo).

    Bidirectional isolation protects layout from unknown-direction text. Top contributors: אורי — 12 edits, سارة — 9 edits, Åsa — 7 edits. Without <bdi>, the numbers can visually attach to the wrong name. A forced override: this sentence renders right-to-left.

    Do-not-translate spans use the translate attribute: the project is called Field Guide in every locale.

    JS-free interactivity

    Exclusive accordion — <details name>

    Why no CSS at all?

    To prove the structure stands on its own. Default browser styling is the shared baseline every user agent — visual or not — understands.

    Why does opening this close the other?

    Both <details> share name="faq", which makes them a mutually exclusive group. No script involved.

    Declarative popover

    Baseline: a feature supported in the current and previous versions of all major browsers. Press Esc or click outside to dismiss — the browser handles it.

    Non-modal dialog

    A <dialog open> renders without any script. Modal behavior (showModal()) needs JavaScript, so this one is non-modal.

    Frontier elements — shown as code, not run

    Three 2026 elements are experimental (Chromium-first, not Baseline) and require JavaScript, so a static page demonstrates them as listings.

    Corrections? File them against the guide — descriptive link text, always: report an issue with the field guide.

    Elements deliberately absent

    <template>, <slot>, <canvas>, and <noscript> serve scripts; <base> rewrites every relative URL on the page (breaking fragment links like the skip link); <object> and <embed> exist for legacy plugin content that <iframe>, <video>, and <img> replaced — all omitted by design from a static document. Deprecated elements (<center>, <font>, <marquee>…) are absent because they are wrong, not because they are old.
